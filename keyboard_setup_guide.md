# คู่มือการติดตั้งและตั้งค่าระบบแป้นพิมพ์เสมือน (Virtual Keyboard) บน Orange Pi CM4

เอกสารนี้สรุปปัญหา วิธีการแก้ปัญหา และขั้นตอนการติดตั้งระบบแป้นพิมพ์เสมือนจริง (On-screen Keyboard / Virtual Keyboard) เพื่อรองรับการใช้งานหน้าจอสัมผัส (Touchscreen) แบบไม่มีคีย์บอร์ดจริง สำหรับส่งต่อให้หัวหน้างานและทีมพัฒนาครับ

---

## 1. ปัญหาที่พบ (Issue)
เนื่องจากตัวหุ่นยนต์ใช้งานผ่าน **หน้าจอสัมผัส DSI Touchscreen** เพียงอย่างเดียว โดยไม่มีการเชื่อมต่อเมาส์หรือคีย์บอร์ดจริง (Physical Keyboard) 
* เมื่อผู้ใช้งานสัมผัสกล่องข้อความ (Input fields) บนหน้าจอเว็บแอปพลิเคชัน (เช่น การป้อนรหัสผ่าน Wi-Fi หรือการพิมพ์ข้อมูลส่วนตัว) **ระบบไม่มีแป้นพิมพ์เสมือนแสดงขึ้นมา** ทำให้ไม่สามารถป้อนข้อมูลใดๆ ได้

---

## 2. วิธีการแก้ไข (Solution)
เราเลือกใช้เครื่องมือ **Onboard** ซึ่งเป็นระบบแป้นพิมพ์เสมือนแบบ System-wide (ระดับระบบปฏิบัติการ) ที่เสถียรและยืดหยุ่นที่สุดสำหรับระบบ Linux Desktop (XFCE / LXDE)

การแก้ไขถูกแบ่งออกเป็น 3 ขั้นตอนหลัก ดังนี้:

### ขั้นตอนที่ 1: ติดตั้งโปรแกรม Onboard
ทำการเชื่อมต่อ SSH เข้าไปยังบอร์ด Orange Pi CM4 แล้วสั่งติดตั้งผ่านตัวจัดการแพ็คเกจ:
```bash
sudo apt-get update
sudo apt-get install -y onboard
```

### ขั้นตอนที่ 2: ตั้งค่าให้เปิดใช้งานอัตโนมัติเมื่อเปิดเครื่อง (Autostart)
เพื่ออำนวยความสะดวกให้แป้นพิมพ์เสมือนรันขึ้นมาพร้อมกับหน้าจอหลักของหุ่นยนต์ทุกครั้งที่บอร์ดบูตระบบ (เมื่อ Autologin เข้าสู่หน้า XFCE):
1. สร้างโฟลเดอร์สำหรับเก็บสคริปต์รันอัตโนมัติ (หากยังไม่มี):
   ```bash
   mkdir -p ~/.config/autostart
   ```
2. คัดลอกการตั้งค่าเริ่มต้นของ Onboard ไปยังโฟลเดอร์รันอัตโนมัติ:
   ```bash
   cp /usr/share/applications/onboard.desktop ~/.config/autostart/
   ```

### ขั้นตอนที่ 3: สั่งให้คีย์บอร์ดทำงานทันที (ไม่ต้องรีบูต)
ในระหว่างดำเนินการเพื่อความต่อเนื่อง สามารถรันคีย์บอร์ดในหน้าจอ Display หลัก `:0` ได้ทันทีผ่านคำสั่ง:
```bash
DISPLAY=:0 nohup onboard >/dev/null 2>&1 &
```

## 3. ทางเลือกเสริม: การเปิดใช้ On-screen Keyboard ในเว็บเบราว์เซอร์ Chromium
หากต้องการให้หน้าแป้นพิมพ์เสมือนถูกควบคุมและแสดงผลเฉพาะในกรอบเบราว์เซอร์เวลาแตะกล่องข้อความบนหน้าเว็บหลัก สามารถเพิ่มพารามิเตอร์นี้ในบรรทัดรันเบราว์เซอร์ของไฟล์ Launch Kiosk:

* **คอนฟิกคำสั่งรันเบราว์เซอร์เดิม:**
  ```text
  DISPLAY=:0 chromium-browser --start-fullscreen --app='file:///home/orangepi/Desktop/robot_home_use/frontend/index.html'
  ```
* **คอนฟิกที่แนะนำเพิ่มเพื่อความสมบูรณ์:**
  ```text
  DISPLAY=:0 chromium-browser --start-fullscreen --enable-virtual-keyboard --touch-events=enabled --app='file:///home/orangepi/Desktop/robot_home_use/frontend/index.html'
  ```
  *(ค่า `--enable-virtual-keyboard` จะบอกให้เบราว์เซอร์แสดงคีย์บอร์ดในตัวเมื่อกดฟิลด์พิมพ์ และ `--touch-events=enabled` จะช่วยให้ตรวจจับระบบสัมผัสจากหน้าจอ DSI ได้แม่นยำขึ้น)*

---

## 4. ผลลัพธ์และการตรวจสอบ (Verification)
1. **เมื่อเริ่มบูตเครื่องใหม่:** หน้าจอจะมีคีย์บอร์ด Onboard แสดงขึ้นมาลอยอยู่บนหน้าจอ XFCE Desktop
2. **การใช้งาน:** ผู้ใช้สามารถลากย้ายขนาดคีย์บอร์ด ย่อเก็บ หรือกดปุ่มตั้งค่าเพื่อปรับ Layout ภาษา และธีมได้อิสระ
3. **การทดสอบ:** เมื่อใช้มือสัมผัสที่กล่องข้อความบนเบราว์เซอร์ ตัวคีย์บอร์ดจะจับพฤติกรรมและป้อนอักขระเข้าไปได้ตามปกติ
