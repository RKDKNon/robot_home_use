# คู่มือการตั้งค่าหน้าจอ DSI Touchscreen บนบอร์ด Orange Pi CM4

เอกสารนี้สรุปปัญหา วิธีวิเคราะห์ และขั้นตอนการแก้ไขหน้าจอ DSI Touchscreen ไม่แสดงผล/ใช้งานสัมผัสไม่ได้ สำหรับส่งต่อให้ทีมและหัวหน้างานครับ

---

## 1. ปัญหาที่พบ (Issue)
หลังจากย้ายระบบมาใช้บอร์ด **Orange Pi CM4** และทำการเชื่อมต่อหน้าจอ DSI Touchscreen ผ่านสายแพ (DSI Ribbon Cable) พบว่า:
* หน้าจอไม่ติด (จอมืด ไม่มีไฟหลังจอ)
* ระบบสัมผัส (Touchscreen) ไม่ทำงาน

---

## 2. วิธีการวิเคราะห์และตรวจสอบ (Diagnostic)
จากการใช้คำสั่งสแกนหาตำแหน่งฮาร์ดแวร์จริงบนบอร์ดผ่านพอร์ตสื่อสารภายใน (I2C Buses):
1. **คำสั่งตรวจสอบ I2C:**
   ```bash
   sudo i2cdetect -y 1
   ```
2. **ผลการสแกน:** พบชิปคุมระบบสัมผัสหน้าจอที่อยู่ตำแหน่ง `0x38` และชิปจอภาพที่อยู่ตำแหน่ง `0x45` บนช่องสัญญาณ **I2C-1**
3. **สาเหตุของปัญหา:** คอนฟิกหน้าจอชุดเดิม (`rpi7-overlay` หรือ `raspi-7inch-touchscreen-dsi0`) พยายามวิ่งไปหาจอที่พอร์ต **I2C-2** หรือ **I2C-3** ซึ่งไม่ตรงกับตำแหน่งพอร์ตจริงบนลายวงจรของ Orange Pi CM4 (ซึ่งเชื่อมต่ออยู่ทางพอร์ต **DSI1** และ **I2C-1**)

---

## 3. ขั้นตอนการแก้ไข (Solution Steps)

ให้ทำตามขั้นตอนแก้ไขคอนฟิกของระบบ Orange Pi CM4 ดังต่อไปนี้:

### ขั้นตอนที่ 1: แก้ไขไฟล์คอนฟิกบูต
ทำการเชื่อมต่อ SSH เข้าไปยัง Orange Pi CM4 แล้วเปิดไฟล์คอนฟิกบูต:
```bash
sudo nano /boot/orangepiEnv.txt
```

### ขั้นตอนที่ 2: แก้ไขบรรทัด Overlay หน้าจอ
ให้ค้นหาบรรทัด `overlays=` และเปลี่ยนไปใช้คอนฟิก **`raspi-7inch-touchscreen`** (ตัวนี้จะทำแผนผัง DSI1 + I2C-1 ให้ตรงกับตำแหน่งหน้าจอจริงโดยอัตโนมัติ) 

* **แก้ไขเป็น:**
  ```text
  overlays=i2c2-m1 i2c3-m0 i2c4-m0 raspi-7inch-touchscreen
  ```

### ขั้นตอนที่ 3: บันทึกและรีบูตระบบ
กด `Ctrl + O` เพื่อบันทึก และ `Ctrl + X` เพื่อออก จากนั้นสั่งรีบูตเครื่อง:
```bash
sudo reboot
```

---

## 4. การตรวจสอบผลลัพธ์ (Verification)

หลังจากเครื่องบูตเสร็จสิ้น สามารถเช็คความสมบูรณ์ของระบบได้ด้วยคำสั่งเหล่านี้:

### 1) ตรวจสอบการเชื่อมต่อจอภาพ (DSI Link)
```bash
dmesg | grep -i dsi
```
**ผลลัพธ์ที่ถูกต้อง:** ควรขึ้นสถานะเชื่อมต่อลิงก์ข้อมูลหน้าจอสำเร็จ:
```text
dw-mipi-dsi-rockchip fe070000.dsi: [drm:dw_mipi_dsi_bridge_enable] final DSI-Link bandwidth: 696 x 1 Mbps
```

### 2) ตรวจสอบการจดจำระบบทัชสกรีน (Touch Input)
```bash
cat /proc/bus/input/devices
```
**ผลลัพธ์ที่ถูกต้อง:** ควรพบคอนโทรลเลอร์ทัชสกรีนขึ้นในระบบ input:
```text
I: Bus=0018 Vendor=0000 Product=0000 Version=0000
N: Name="fts_ts"
P: Phys=
S: Sysfs=/devices/platform/fe5a0000.i2c/i2c-1/1-0038/input/input1
H: Handlers=event1
```
*(ถ้าขึ้นตามนี้แปลว่าระบบทัชสกรีนและหน้าจอ DSI เริ่มต้นทำงานได้สมบูรณ์แล้วครับ)*
